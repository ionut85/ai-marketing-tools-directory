# AI x Marketing Tools Directory - Replit Agent Prompt

## Project Overview

Build a minimal, fast directory website for AI x Marketing tools. Design inspired by Vercel's template gallery - clean, spacious, black/white aesthetic with prominent search.

**Primary Goal:** Users should see MOST tools on first load. Compact cards, no pagination - filtering shows/hides tools dynamically.

**Stack:**
- Next.js 15 (App Router)
- Tailwind CSS 4
- shadcn/ui
- TypeScript
- OpenAI API for natural language search
- JSON files for data (no database)
- Deploy: Vercel

---

## Core Pages

### 1. Homepage / Directory (/)

**Layout (top to bottom):**

- Minimal header: Logo on left, "Landscape" and "About" links on right
- Hero section: Large centered H1 "Discover AI x Marketing Tools" with subtitle
- LARGE search bar (48px+ height) centered below hero
- Main content: Filter sidebar on LEFT, tool grid on RIGHT
- Tool grid: 4-6 columns of compact cards
- Footer: "Submit a Tool" link

**Search Bar Behavior:**
- Large, centered, prominent (48px height minimum)
- Placeholder: Search tools... try "AI for ad creative" or "attribution platforms"
- On type: filters results in real-time (client-side for speed)
- Natural language via OpenAI: understands "show me creative tools" or "what helps with measurement"
- Debounced API call (300ms) for AI-powered semantic search

**Tool Cards (COMPACT - this is key):**
- Card size: ~180-220px wide, ~100-120px tall
- Logo: 32x32px with fallback (colored circle + initial)
- Tool name next to logo
- Tagline: 1 line max, truncated
- 2-3 small badge tags
- Hover: subtle shadow, cursor pointer
- Click -> tool detail page

**Filter Sidebar (Left):**
- Collapsible sections: "Category", "Use Case", "Pricing"
- Checkboxes with counts: "Creative (23)"
- Multi-select (AND logic for categories, OR for use cases)
- "Clear all" link
- On mobile: slide-out sheet

**Results Counter:**
- Bottom of grid or top-right: "Showing 47 of 103 tools"
- Updates live as filters change

## Key Implementation Details

### Compact Tool Cards

```tsx
// Target: 180-220px wide, ~100-120px tall
<Card className="p-3 hover:shadow-md transition-shadow cursor-pointer">
  <div className="flex items-center gap-2 mb-1">
    <Image src={logo} alt="" className="w-8 h-8 rounded" />
    <span className="font-medium text-sm truncate">{name}</span>
  </div>
  <p className="text-xs text-muted-foreground line-clamp-1">{tagline}</p>
  <div className="flex gap-1 mt-2">
    <Badge variant="secondary" className="text-xs">{category}</Badge>
  </div>
</Card>
```

### Search with Debounce

```tsx
const [query, setQuery] = useState('')
const debouncedQuery = useDebounce(query, 300)

useEffect(() => {
  if (debouncedQuery.length > 2) {
    fetch('/api/search', {
      method: 'POST',
      body: JSON.stringify({ query: debouncedQuery })
    }).then(/* update results */)
  }
}, [debouncedQuery])
```

### Filter Logic

- Client-side filtering for speed
- Checkboxes update URL params: ?category=creative&useCase=copywriting
- Grid re-renders with CSS transitions (fade out/in)

## Notes for Agent

- Use shadcn components where possible (Input, Badge, Card, Checkbox, Button)

- Filter sidebar is essential - don't skip it
